# HireSight

HireSight is a light-weight web application that helps job seekers gauge the strength of their resumes against a target role. Users upload a PDF resume, supply details about the job posting, and the app returns an **ATS compatibility score**, breakdown of content, tone, structure, and personalized improvement tips generated by an AI service.

The project uses [Puter.js](https://puter.js/) for authentication, file storage, key‑value persistence and AI calls, with a React‑in‑Vite frontend written in TypeScript and styled with TailwindCSS.

---

## 🚀 Key Features

- **Resume upload** (PDF only) with drag‑and‑drop UI.
- **Automatic PDF→image conversion** for previewing first page.
- **AI‑powered analysis** that returns:
  - Overall score (0‑100)
  - ATS suitability
  - Tone & style, content, structure, and skills breakdown
  - A handful of “good”/“improve” tips for each category
- **Review past analyses** (stored in a simple KV).
- Lightweight Dockerfile for containerised deployment.

## 🧱 Architecture Overview

1. **Frontend** – a React app using React Router. Routes include:
   - `/upload` — main form for submitting resumes
   - `/resume/:id` — dashboard showing a specific analysis with a screenshot, score badge, and tips
   - auth flow handled via Puter.js (sign in/out links in `Navbar`)
2. **Puter.js services** – available globally in `window.puter`:
   - `fs` for uploading/resuming files
   - `kv` for persisting metadata and results
   - `ai` for sending prompts to the AI model and receiving JSON feedback
3. **Utilities**
   - `pdf2img.ts` converts the first page of a PDF to an image using pdf.js
   - `utils.ts` includes helpers such as UUID generation
   - `routes` folder contains individual page components

## 🛠️ Local Development

Prerequisites:

- Node 20+ (the Dockerfile uses `node:20-alpine`)
- Optional: Docker if you plan to run the container

Steps:

```bash
# clone the repo
git clone https://github.com/<your‑user>/HireSight.git
cd HireSight

# install dependencies
npm install

# create a .env file (see Environment section below)
```

Start the dev server:

```bash
npm run dev
```

The app will be available at `http://localhost:5173`. HMR is enabled so changes are reflected immediately.

## 🧩 Environment Variables

The frontend reads configuration from Vite's `import.meta.env`. For example:

```env
VITE_PUTER_API_KEY=...
VITE_PUTER_BASE_URL=https://...   # if you self‑host Puter services
```

Create a `.env` file in the project root (it’s already ignored by `.gitignore`). The exact variables depend on how you deploy the Puter runtime and which AI provider you’re using; consult Puter.js documentation.

## 📦 Production Build & Docker

To build a production bundle:

```bash
npm run build
```

This emits a `build/` directory that can be served with a static file server or packaged with the provided Dockerfile.

**Using Docker:**

```bash
# build image
docker build -t hiresight .

# run container on port 3000
docker run -p 3000:3000 hiresight
```

The server listens on port 3000 by default and serves the built assets. You can also deploy the same image to any Docker‑compatible host.

## ✅ Usage

1. Open the app in a browser.
2. If authentication is enabled, sign in via the navbar.
3. Fill in the job title, company name and paste (or upload) the job description.
4. Choose a PDF resume file and click **Analyze Resume**.
5. Wait a few seconds while the file is uploaded, converted and sent to the AI.
6. You’ll be redirected to a results page showing your score, a preview of the resume first page, and improvement tips.
